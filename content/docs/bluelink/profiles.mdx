The Bluelink Manager supports **profiles** that configure which CLI binary and plugins are installed. Profiles allow the same manager binary and shared components to serve different toolchains.

## What is a Profile?

A profile defines:
- Which CLI binary to install (and from which repository)
- Which set of Deploy Engine plugins to include as defaults

Shared components (Deploy Engine and Blueprint Language Server) are installed once regardless of which profiles are active. The manifest at `~/.bluelink/manifest.json` tracks which profiles are currently installed.

## Available Profiles

| | Bluelink (default) | Celerity |
|---|---|---|
| **CLI binary** | `bluelink` | `celerity` |
| **CLI source** | [newstack-cloud/bluelink](https://github.com/newstack-cloud/bluelink) | [newstack-cloud/celerity](https://github.com/newstack-cloud/celerity) |
| **Core plugins** | `newstack-cloud/aws` | `newstack-cloud/aws`, `newstack-cloud/celerity` |
| **Deploy Engine** | Shared | Shared |
| **Blueprint Language Server** | Shared | Shared |

For Celerity CLI downloads, SBOMs, and verification, see the [Celerity installation guide](https://celerityframework.io/docs/framework/installing-celerity).

## Installing a Profile

### Default Installation (Bluelink profile)

When you install using the Bluelink Manager without specifying a profile, the Bluelink profile is used:

```bash
bluelink-manager install
```

This is equivalent to:

```bash
bluelink-manager install --profile bluelink
```

### Installing the Celerity Profile

To install the Celerity profile:

```bash
bluelink-manager install --profile celerity
```

### Installing Both Profiles

Both profiles can be installed side-by-side on the same machine:

```bash
bluelink-manager install --profile bluelink,celerity
```

Shared components (Deploy Engine and Blueprint Language Server) are only downloaded once. Each profile's CLI binary and plugins are installed independently.

## Managing Profiles

### Updating a Specific Profile

```bash
# Update all installed profiles
bluelink-manager update

# Update only the Celerity profile
bluelink-manager update --profile celerity
```

### Uninstalling a Profile

```bash
# Uninstall only the Celerity profile
bluelink-manager uninstall --profile celerity
```

When you uninstall a single profile, shared components remain intact if another profile still needs them. Only the profile-specific CLI binary and plugins are removed.

### Checking Status

```bash
bluelink-manager status
```

This shows all installed profiles, component versions, and the Deploy Engine service status.

## Coexistence

Profiles coexist cleanly on the same machine. Installing the Celerity profile alongside Bluelink does not affect the Bluelink CLI or its configuration. The `bluelink` and `celerity` binaries are separate executables that share the same Deploy Engine.

Both CLIs communicate with the same Deploy Engine instance, which manages plugins from all installed profiles. The Blueprint Language Server is also shared and provides diagnostics, validation and auto-complete for blueprint files in editors through extensions such as the [Bluelink VSCode extension](https://marketplace.visualstudio.com/items?itemName=NewStack.bluelink).
