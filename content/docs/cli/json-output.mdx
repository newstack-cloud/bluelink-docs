This document provides detailed examples of the JSON output format for Bluelink CLI commands when using the `--json` flag.
JSON output mode is useful for CI/CD pipelines, scripting, and programmatic tooling consumption.

When `--json` is enabled, the CLI outputs a single JSON object to stdout when the operation completes.
This implies non-interactive mode (no TUI, no streaming text output).

## Stage Command

The `bluelink stage` command outputs JSON describing the staged changes, including new resources, resource changes, child blueprint changes, exports, and a summary.

#### Reconciliation Fields

| Field | Description |
|-------|-------------|
| `reconciliation.instanceId` | The ID of the blueprint instance with drift |
| `reconciliation.resources` | Array of resources that have drifted or are in interrupted state |
| `reconciliation.resources[].resourceId` | The unique identifier of the drifted resource |
| `reconciliation.resources[].resourceName` | The logical name of the resource in the blueprint |
| `reconciliation.resources[].resourceType` | The type of the resource (e.g., `aws/sqs/queue`) |
| `reconciliation.resources[].childPath` | Path to child blueprint if resource is in a child (empty for root) |
| `reconciliation.resources[].type` | Type of issue: `drift` or `interrupted` |
| `reconciliation.resources[].externalState` | Current state of the resource in the cloud provider |
| `reconciliation.resources[].persistedState` | State stored in the deploy engine |
| `reconciliation.resources[].changes` | Detailed field-level changes between blueprint and external state |
| `reconciliation.resources[].resourceExists` | Whether the resource exists in the cloud provider |
| `reconciliation.resources[].recommendedAction` | Suggested action: `accept_external`, `update_status`, or `manual_cleanup_required` |
| `reconciliation.links` | Array of links that have drifted |
| `reconciliation.hasInterrupted` | Whether any resources are in interrupted state |
| `reconciliation.hasDrift` | Whether drift was detected in any resources |
| `reconciliation.hasChildIssues` | Whether issues exist in child blueprints |

### Error (Validation Failure)

When blueprint validation fails during staging:

```json
{
  "success": false,
  "error": {
    "type": "validation",
    "message": "Blueprint validation failed",
    "diagnostics": [
      {
        "level": "ERROR",
        "message": "Unknown resource type 'aws/invalid/type'",
        "line": 15,
        "column": 5
      }
    ]
  }
}
```

### Error (Instance Not Found)

When the specified instance does not exist:

```json
{
  "success": false,
  "error": {
    "type": "not_found",
    "message": "Blueprint instance 'my-nonexistent-stack' not found"
  }
}
```

#### Deploy Output Fields

| Field | Description |
|-------|-------------|
| `success` | Whether the deployment completed successfully |
| `instanceId` | The unique identifier of the deployed blueprint instance |
| `instanceName` | The user-defined name of the blueprint instance |
| `changesetId` | The ID of the changeset that was deployed |
| `status` | The deployment status (e.g., `DEPLOYED`, `DEPLOY_FAILED`) |
| `instanceState` | The complete state of the blueprint instance after deployment |
| `instanceState.resources` | Map of resource IDs to their deployed state including `specData` |
| `instanceState.exports` | The resolved export values from the blueprint |
| `instanceState.childBlueprints` | Nested child blueprint instances with their own resources and exports |
| `instanceState.durations` | Timing information for the deployment |
| `summary.successful` | Count of successfully deployed elements |
| `summary.failed` | Count of failed elements |
| `summary.interrupted` | Count of interrupted elements |
| `summary.elements` | Array of all deployed elements with their status and action |

### Deployment Drift Detected

When using `--stage` with the deploy command and drift is detected during the staging process, the CLI will return a drift response instead of proceeding with deployment. This indicates that external changes have been made to resources and reconciliation is required before staging changes.

```json
{
```
